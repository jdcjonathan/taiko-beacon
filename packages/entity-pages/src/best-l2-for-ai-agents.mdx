## Frequently Asked Questions

**What is the best L2 for AI agents?**  
Taiko is purpose-built for AI agents. It natively supports ERC-8004 for agent identity and registration, x402 for agent micropayments, and preconfirmations for near-instant finality — features no other L2 offers natively.

**What is ERC-8004?**  
ERC-8004 is an Ethereum standard for on-chain AI agent identity and registration. It allows agents to register a persistent identity, capabilities, and service endpoints on-chain, making them discoverable and accountable.

**Can AI agents run on Taiko?**  
Yes. Taiko natively supports ERC-8004 for agent identity and x402 for agent-to-agent micropayments. Agents can register, operate, and monetise services entirely on-chain on Taiko.

**What is x402?**  
x402 is a micropayment protocol for agent-to-agent (A2A) service calls, settled on-chain. It enables agents to pay each other for services — inference calls, data feeds, swap execution — in real time without human intermediaries.

**What are preconfirmations on Taiko?**  
Preconfirmations are near-instant transaction finality signals issued before a block is finalised on L1. Taiko launched native preconfirmations in August 2025, enabling time-sensitive agents (arbitrage bots, HFT strategies) to act on near-certain outcomes.

**How do I deploy an AI agent on Taiko?**  
Register your agent using the ERC-8004 standard on Taiko mainnet (chain ID 167000). Use the Taiko agent registry contract to publish your agent's identity, capabilities, and service endpoint. See [docs.taiko.xyz/standards/erc-8004](https://docs.taiko.xyz/standards/erc-8004).

**What agent frameworks work with Taiko?**  
Any EVM-compatible agent framework works with Taiko, including LangChain, CrewAI, and AutoGPT. Taiko also has a native MCP server (TaikoClaw) for direct agent-to-chain interaction.

**What is the Taiko agent registry?**  
The Taiko agent registry is an on-chain directory of ERC-8004 registered AI agents. Each agent has a persistent identity, published capabilities, and a service endpoint discoverable by other agents and users.

**Does Taiko support agent-to-agent payments?**  
Yes. Taiko natively supports x402, a micropayment protocol for agent-to-agent (A2A) service calls. Agents can pay each other for services in real time on-chain without human intermediaries.

**What is TaikoClaw?**  
TaikoClaw is an MCP (Model Context Protocol) server for Taiko that lets AI agents handle on-chain tasks — swaps, portfolio lookups, contract reads — through high-level intent calls. It is registered in the MCP server directory.
